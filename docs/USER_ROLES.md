# User roles V1

## Communication

Rôle principal de la V1.

La communication peut :

* créer et modifier les événements ;
* suivre les démarches ;
* uploader et classer les documents ;
* gérer les tâches ;
* gérer la PLV ;
* préparer les kits communication ;
* sélectionner les formations à pousser ;
* créer les landing pages ;
* suivre les leads ;
* préparer les relances ;
* compléter le reporting ;
* analyser le ROI.

## Direction

La direction consulte :

* le dashboard ;
* les événements à arbitrer ;
* les coûts ;
* les décisions ;
* les retours ;
* le ROI ;
* les recommandations de reconduction.

## Finance

La finance consulte principalement :

* devis ;
* factures ;
* budgets estimés ;
* coûts réels ;
* documents financiers.

## Chargées de formation

Les chargées de formation sont utilisatrices secondaires en V1.

Elles peuvent avoir besoin de :

* consulter les événements qui les concernent ;
* voir les briefs ;
* consulter les formations à pousser ;
* ajouter des retours terrain ;
* consulter ou suivre certains leads ;
* voir les documents utiles ;
* identifier les prochaines actions.

Une interface simplifiée pour elles sera étudiée plus tard.

## Équipes terrain

Les équipes mobilisées sur événements consultent :

* brief événement ;
* horaires ;
* lieu ;
* équipe prévue ;
* argumentaire ;
* formations prioritaires ;
* documents utiles ;
* QR codes / landing pages ;
* PLV à prendre.

## Prospects

Les jeunes, parents ou prospects n’ont pas accès à la plateforme interne.

Ils accèdent uniquement à :

* landing pages publiques ;
* formulaires publics ;
* pages d’information ;
* inscriptions JPO ;
* demandes de rappel ;
* demandes de rendez-vous.

## Employeurs

Les employeurs n’ont pas d’espace dédié en V1.

Un portail employeur pourra être envisagé en V2 ou dans un projet plus large.
